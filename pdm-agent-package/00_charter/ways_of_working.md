# Ways of Working（Lean × Agile / Dual-Track）

## 1. ループ
- 仮説 → 最小実験 → 計測 → 学習 → 意思決定 → 反映

## 2. Dual-Track（並走）
- Discovery: 「検証されたバックログ」を生成する
- Delivery: 「リリース可能なソフト」を生成する

## 3. コミットメント（Scrum）
- Product Backlogのコミットメント：Product Goal
- Sprint Backlogのコミットメント：Sprint Goal
- Incrementのコミットメント：Definition of Done（DoD）

## 4. 守るルール（最小）
- 同時に走る実験は最大2本
- 検証されていない大規模実装はしない（“学習”が目的）
- 計測なしリリースは禁止（イベント/ログの最低限）
- Decision Logは必ず残す（なぜやめた/続けたか）
